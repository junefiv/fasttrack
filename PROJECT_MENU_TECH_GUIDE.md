# FASTTRACK 프로젝트 메뉴 종합 분석 문서

## 1. 문서 목적

이 문서는 현재 FASTTRACK 프로젝트의 전체 메뉴 구조와 각 메뉴별 서비스 기능, 개발 아키텍처, 기술 스택, UI/UX 패턴, 데이터 흐름, 접근성 및 반응형 설계를 한 번에 파악할 수 있도록 정리한 기술 문서입니다.

## 2. 범위

### 포함

- 웹 애플리케이션의 실제 라우팅과 사이드바 메뉴 구조
- 각 메뉴별 화면 구성과 상호작용 정의
- React 프런트엔드 구조와 Supabase 기반 데이터 흐름
- AI/Gemini 연동, YouTube 플레이어, PDF 교재 연동 구조
- 현재 구현 상태와 코드 기준 개선 우선순위

### 제외

- 데이터베이스 전체 ERD의 완전한 스키마 명세
- 디자인 시안 파일, 피그마 원본, 배포 인프라 구성
- 실제 운영 환경의 권한 정책, 인증 정책, 분석 도구 상세 설정
- 기획 확정 문서가 없는 공식 사업 로드맵

## 3. 작성 기준 및 가정

- 본 문서는 `c:\FASTTRACK` 기준 현재 코드베이스만 분석했습니다.
- “향후 개선 계획”은 코드에 드러난 제약, 플레이스홀더 상태, 구현 패턴을 기반으로 도출한 권고안입니다.
- 일부 지표는 실데이터가 아니라 데모 데이터 또는 브라우저 로컬 상태를 사용합니다.
- 메뉴 노출 여부는 사이드바 기준과 실제 라우팅 기준을 함께 표기했습니다.

## 4. 전체 프로젝트 개요

FASTTRACK는 학습 대시보드, 강의/교재 학습, 모의고사/문제은행/드릴, AI 코치 기능을 통합한 학습 플랫폼입니다.  
구조상 크게 다음 두 축으로 나뉩니다.

- **Study Room**: 강의, 교재, 모의고사, 문제은행, 드릴, 학습 아카이브
- **D-Agent**: 학습코치, 커리큘럼 코치, 입시코치

## 5. 전체 메뉴 구조도

```text
FASTTRACK
├─ 메인 대시보드 (/)
├─ STUDY ROOM
│  ├─ 인강/교재 학습 (/study/videos)
│  │  └─ 강의 시청 상세 (/study/videos/watch/:sessionId)
│  ├─ 모의고사 & 드릴 (/study/mock-exam)
│  │  ├─ 문제은행 (/study/mock-exam/bank)
│  │  ├─ 카탈로그 미리보기 (/study/mock-exam/preview/:catalogId)
│  │  ├─ 모의고사 응시 (/study/mock-exam/mock/:examId)
│  │  ├─ 모의고사 결과 (/study/mock-exam/mock/:examId/result/:resultId)
│  │  └─ 드릴 응시 (/study/mock-exam/drill?ids=...)
│  └─ 학습 아카이브 (/study/archive)
├─ D-AGENT
│  ├─ 학습코치 (/d-agent/mh-chat)
│  ├─ 네비게이터 = 커리큘럼 코치 (/d-agent/learning-coach)
│  └─ 입시코치 (/d-agent/admission-coach)
└─ 보조/숨김 라우트
   ├─ 레거시 영상 리다이렉트 (/videos, /videos/watch/:sessionId)
   ├─ 이북 보기 플레이스홀더 (/ebook)
   └─ 이북 업로드 운영 화면 (/ebook/upload, env 플래그 필요)
```

## 7. 전체 개발 아키텍처

### 7.1 프런트엔드 구조

- **프레임워크**: React 19 + TypeScript + Vite
- **라우팅**: React Router DOM
- **UI 라이브러리**: Mantine Core, Mantine Hooks
- **스타일링**: 페이지/기능별 CSS 파일 분리
- **상태 관리**: 페이지 로컬 상태 중심의 훅 기반 구조
- **공통 패턴**:
  - 페이지는 `src/pages`
  - 기능 단위 컴포넌트는 `src/features`
  - 재사용 가능 UI는 `src/components`
  - 데이터/유틸은 `src/lib`

### 7.2 백엔드 및 데이터 계층

- **BaaS**: Supabase
- **DB 접근 방식**: 프런트엔드에서 `@supabase/supabase-js`로 직접 조회/저장
- **주요 데이터 도메인**
  - 강의: `subjects`, `lectures`, `lecture_sessions`, `lecture_captions`
  - 교재: `learning_resources`, `ebook_pages`
  - 시험/문항: `fasttrack_mock_exams`, `fasttrack_problems`, `fasttrack_mock_exam_catalog`, `fasttrack_mock_exam_catalog_problems`
  - 결과/통계: `fasttrack_test_results`, `fasttrack_user_answers`, `fasttrack_student_stats`, `fasttrack_drill_problems`

### 7.3 AI 아키텍처

- **모델**: Gemini API
- **사용 영역**
  - 강의/교재 질문 응답
  - 학습코치 멀티턴 채팅
  - 커리큘럼 코치의 강점·취약점 분석
- **컨텍스트 구성**
  - 전체 강의 자막
  - 교재 전체 본문
  - PDF 하이라이트
  - 페이지 단위 RAG + PDF 청크 RAG

### 7.4 교재 추출 파이프라인

```text
사용자 PDF 업로드
→ Supabase Storage 저장
→ learning_resources 행 생성
→ DB 트리거 / Webhook
→ Supabase Edge Function extract-ebook-pages 호출
→ PDF 텍스트 추출
→ ebook_pages 적재
→ 강의 상세에서 PDF 뷰어 + RAG + 질문 기능에 활용
```

## 9. 공통 UI/UX 패턴

### 9.1 내비게이션 패턴

- 좌측 고정 사이드바 + 우측 콘텐츠 영역
- `STUDY ROOM`, `D-AGENT` 아코디언 섹션
- 모바일에서는 햄버거 버튼으로 사이드바 열기/닫기
- 시험/드릴 응시 화면은 포커스 모드로 사이드바 숨김

### 9.2 화면 구성 패턴

- 상단 헤더 + 카드/리스트/패널 구조
- 목록 → 상세 → 실행 흐름이 명확한 계층형 설계
- Drawer를 활용한 보조 컨텍스트 제공
  - 자막 검색
  - 질문하기
  - 교재 보기
  - 모바일 본문 보기

### 9.3 상호작용 패턴

- 버튼, 탭, 드로어, 검색, 필터, 카드 클릭
- 진행 상태 표시
  - 로딩
  - 빈 상태
  - 오류 상태
  - 처리 중 상태
- 사용자 선택 상태 시각화
  - 활성 탭
  - 현재 문항
  - 완료 문항
  - 플래그 문항

### 9.4 접근성 공통 패턴

- `aria-label`, `aria-live`, `aria-expanded`, `aria-controls`, `role="tablist"`, `role="tabpanel"` 적용
- 키보드 접근 가능한 카드형 버튼 일부 구현
- 상태 안내 텍스트와 에러 메시지 제공
- 다만 색상 대비, 포커스 링 일관성, 스크린리더 전용 설명의 정교화는 추가 개선 여지 존재

## 10.1 메인 대시보드

### 서비스 개요

- 학습 전반의 핵심 신호를 한 화면에서 요약하는 진입 페이지
- 현재는 실데이터 연동보다는 컨셉 검증용 카드형 대시보드 성격이 강함

### 경로

- `/`

### 주요 기능 리스트

- 추천 드릴 CTA 제공
- 목표 대학 대비 현재 위치 브리핑 카드
- FOMO 기반 비교 자극 카드
- 이탈 경고 카드

### 사용 프레임워크/라이브러리

- React Router `Link`
- 자체 CSS 기반 레이아웃

### 데이터 흐름도

```text
사용자 진입
→ 정적 카드 렌더링
→ 특정 CTA 클릭
→ 모의고사 & 드릴 화면으로 이동
```

### 화면 레이아웃 설명

- 상단: 섹션 번호, 타이틀, 서브타이틀
- 본문: 4개 카드 그리드

### 상호작용 요소별 동작 정의

- “모의고사 & 드릴 열기” 클릭 시 관련 허브로 이동
- 나머지 카드는 정보 표시 중심이며 직접 액션 없음

### 반응형 디자인 적용 여부

- 카드 그리드 기반으로 축소 대응 가능
- 다만 모바일에서 정보 밀도 최적화는 제한적

### 접근성 고려사항

- 링크 CTA 명확
- 정보 카드 구조는 단순해 읽기 쉬움
- 실시간 수치 업데이트나 상태 변경에 대한 추가 ARIA 처리 없음

### 개발 현황

- **상태**: 데모형 구현 완료
- **특징**: 실데이터 미연동

### 향후 개선 계획

- 실제 학습 통계 연동
- 카드 클릭 시 상세 drill-down 연결
- 개인별 우선순위 기반 추천 재정렬

## 10.2 인강/교재 학습

### 서비스 개요

- 과목 → 강사 → 강좌 → 회차 구조로 강의를 탐색하는 메뉴
- 강의 시청 상세 화면과 직접 연결되는 탐색 허브

### 경로

- 목록: `/study/videos`
- 상세: `/study/videos/watch/:sessionId`

### 주요 기능 리스트

- 과목 탭 전환
- 강사 목록 탐색
- 강좌 목록 탐색
- 회차 목록 탐색
- 썸네일 기반 강의 진입

### 사용 프레임워크/라이브러리

- Mantine `Tabs`, `Paper`, `Image`, `Badge`, `Button`, `Alert`, `Skeleton`
- Supabase 직접 조회

### 데이터 흐름도

```text
페이지 진입
→ subjects / lectures / lecture_sessions 병렬 조회
→ 과목 선택
→ 해당 과목의 강사 목록 계산
→ 강사 선택
→ 해당 강사의 강좌 목록 계산
→ 강좌 선택
→ 회차 목록 표시
→ 회차 클릭
→ 강의 시청 상세 이동
```

### 화면 레이아웃 설명

- 상단: 페이지 제목
- 중단: 과목 탭
- 본문: 3단계 drill-down
  - 강사 목록
  - 강좌 목록
  - 회차 목록

### 상호작용 요소별 동작 정의

- 과목 탭 클릭: 현재 drill 단계 초기화
- 강사 카드 클릭/Enter/Space: 강좌 목록 이동
- 강좌 카드 클릭/Enter/Space: 회차 목록 이동
- 회차 카드 클릭: 상세 플레이어 이동
- 뒤로 버튼: 이전 탐색 단계 복귀

### UI/UX 디자인 패턴

- 단계적 drill-down 탐색
- 카드형 선택
- 목록 깊이에 따라 상단 “뒤로” 제공
- 로딩 시 Skeleton, 실패 시 Alert, 빈 상태 메시지 제공

### 반응형 디자인 적용 여부

- Mantine Stack/Group 기반이라 기본 대응 가능
- 그러나 대규모 데이터셋의 모바일 탐색 최적화는 추가 작업 필요

### 접근성 고려사항

- 탭 구조와 버튼 구조 명시적
- 일부 카드에 키보드 조작 지원
- 이미지 `alt=""` 처리로 장식 이미지 성격 유지

### 개발 현황

- **상태**: 핵심 기능 구현 완료
- **제약**: 검색, 정렬, 즐겨찾기, 진도 기준 필터는 없음

### 향후 개선 계획

- 강의 검색/정렬 추가
- 최근 시청/이어보기 표시
- 회차별 시청률/완료 상태 반영

## 10.3 강의 시청 상세

### 서비스 개요

- YouTube 강의 재생, 실시간 자막 확인, 자막 검색, 교재 열람, AI 질문을 하나의 화면에서 제공하는 핵심 학습 화면

### 경로

- `/study/videos/watch/:sessionId`

### 주요 기능 리스트

- YouTube 영상 재생 및 시크
- 현재 구간 자막 표시
- 자막 검색 및 검색 결과 시점 이동
- 교재 PDF 보기
- 교재 내 텍스트 선택 기반 질문 생성
- 회차 전체 자막 + 교재 본문 기반 AI 질의응답
- 복수 질문 스레드 유지

### 사용 프레임워크/라이브러리

- Mantine `Drawer`, `Paper`, `ScrollArea`, `Button`, `Group`, `Stack`, `Text`
- YouTube IFrame API
- `react-pdf`
- Gemini API
- Supabase

### 데이터 흐름도

```text
세션 진입
→ lecture_sessions 조회
→ lecture_captions / learning_resources / ebook_sections 조회
→ YouTube 플레이어 로드
→ 현재 재생 시각 기준 active caption 계산

질문하기 흐름
→ 질문 스레드 생성
→ 전체 자막 + 교재 본문 + RAG 발췌 결합
→ Gemini 호출
→ 응답 저장

교재 흐름
→ learning_resources 조회
→ PDF 렌더링
→ 텍스트 드래그
→ 선택 텍스트 + 페이지번호 + pdf_url 전달
→ 질문 Drawer에서 교재 맥락 질문 생성
```

### 화면 레이아웃 설명

- 상단: breadcrumb
- 본문 상단: 비디오 플레이어
- 중간: 현재 자막 카드 + 질문하기 버튼
- 하단: 재생 컨트롤, 자막 검색 버튼, 교재 보기 버튼, 시크 바
- 우측 Drawer:
  - 자막 검색
  - 질문하기
  - 교재 보기

### 상호작용 요소별 동작 정의

- 재생/일시정지 버튼: 플레이어 상태 토글
- 진행 바 드래그: 특정 시점 시크
- 자막 검색 버튼: 우측 Drawer 오픈
- 자막 행 클릭: 해당 시점 이동
- 검색 일치 행에서 질문하기: 해당 시점 기준 질문 스레드 생성
- 교재 보기 버튼: PDF Drawer 토글
- PDF 텍스트 선택: 선택된 텍스트 기준 질문 스레드 생성
- 질문하기 버튼: 현재 재생 시각 기준 질문 스레드 생성

### UI/UX 디자인 패턴

- 학습 컨텍스트 보존형 멀티패널 UI
- 핵심 시청 화면을 방해하지 않는 Drawer 기반 부가 기능 분리
- 컨텍스트 앵커링
  - 영상 질문: 재생 시각 ±10분
  - 교재 질문: 하이라이트 페이지 ±10페이지

### 반응형 디자인 적용 여부

- Drawer 중심 UI라 협소한 화면에서도 기능 사용 가능
- 플레이어는 16:9 비율 유지
- 일부 고밀도 기능은 데스크톱 중심 최적화

### 접근성 고려사항

- 현재 자막에 `aria-live` 적용
- Drawer 제목과 breadcrumb 제공
- 자막 검색 입력/버튼/목록 구조 명확
- PDF 선택 기반 질문 기능은 포인팅 디바이스 친화적이며 키보드 대안은 제한적

### 개발 현황

- **상태**: 프로젝트 내 가장 고도화된 메뉴 중 하나
- **구현 범위**: 비디오, 자막, PDF, AI 질문의 통합 경험 제공
- **제약**: 시청 로그 영속화, 북마크, 노트 작성, 진도 동기화 미구현

### 향후 개선 계획

- 자동 진도 저장 및 이어보기
- 자막 북마크/하이라이트
- 질문 스레드 서버 저장
- 교재/자막/노트의 통합 학습 타임라인 제공

## 12. 반응형 디자인 적용 현황

### 적용이 명확한 영역

- 전체 앱 셸: 900px 이하 모바일 사이드바 전환
- 모의고사 응시: 좁은 화면에서 본문 Drawer 분리
- 강의 상세: Drawer 중심 보조 UI
- YouTube 플레이어: 고정 비율 반응형

### 보완이 필요한 영역

- 학습코치 2열 구조의 모바일 최적화
- 커리큘럼 코치의 고밀도 차트 영역
- 문제은행의 다중 필터 UI

## 13. 접근성 종합 평가

### 강점

- 아코디언, 탭, 패널, 드로어에 필요한 ARIA 속성 적용이 비교적 잘 되어 있음
- 주요 상태 메시지와 오류 메시지 제공
- 로딩/비어 있음/오류 상태가 분리되어 인지 가능

### 보완 과제

- 포커스 이동과 키보드 순서의 세밀한 점검 필요
- 카드형 클릭 요소의 일관된 키보드 지원 확대 필요
- PDF 텍스트 선택 기반 질문 기능의 비마우스 대안 필요
- 색상 중심 상태 표시의 텍스트 보조 강화 필요

## 14. 기술적 강점

- 메뉴 구조와 라우팅이 명확하고 확장 가능
- 학습 흐름이 “탐색 → 실행 → 결과 → 후속 학습”으로 잘 연결됨
- 공통 렌더러(`ProblemRenderer`)로 문제 UI 일관성 유지
- 강의 상세는 자막/교재/AI를 결합한 고부가가치 학습 경험 제공
- Supabase + Edge Function + Gemini 조합으로 전체 서비스 체인이 유기적으로 연결됨

## 15. 핵심 리스크

- 인증/권한 모델이 문서화되지 않아 사용자 식별이 제한적
- 일부 지표가 데모 데이터 기반이라 서비스 신뢰성 해석 주의 필요
- 프런트엔드 직접 DB 접근 비중이 높아 권한 설계가 매우 중요
- 이북 업로드는 운영 편의성은 높지만 보안상 보강 필요
- 메뉴별 테스트 코드와 자동 검증 체계가 아직 부족함

## 16. 우선순위 기반 개선 권고

### MVP 안정화 우선

1. 학습코치/커리큘럼 코치의 사용자 식별 및 서버 저장 구조 정리
2. 모의고사/드릴 응시 중간 저장 및 복구
3. 업로드/AI 기능의 권한 및 보안 정책 강화

### 사용자 가치 확장

1. 학습 아카이브 정식 구현
2. 입시코치 초기 버전 오픈
3. 강의 상세의 노트/북마크/질문 기록 영속화

### 운영 효율화

1. 메뉴별 사용 로그 수집
2. 오류 추적과 분석 이벤트 도입
3. 실데이터/데모데이터 혼재 구간 정리

## 17. 오픈 이슈

- 실제 사용자 인증 체계가 어디까지 구현되었는지 추가 확인 필요
- 커리큘럼 코치의 데모 지표를 실데이터로 언제 전환할지 결정 필요
- 학습 아카이브와 입시코치의 제품 정의가 아직 비어 있음
- 이북 보기 사용자 메뉴와 운영 업로드 메뉴의 역할 분리가 필요

## 18. 결론

현재 FASTTRACK는 **강의 학습**, **모의고사/드릴**, **AI 코치** 세 영역에서 이미 유의미한 프로토타입을 넘어선 구조를 갖추고 있습니다. 특히 강의 상세 화면과 모의고사-드릴 연계 흐름은 서비스 핵심 가치가 잘 드러나는 영역입니다. 반면 **학습 아카이브**, **입시코치**, **사용자 영속화**, **보안/운영 체계**는 다음 단계에서 반드시 정리해야 할 핵심 과제입니다.

즉, 이 프로젝트는 “개별 기능 데모” 단계를 지나 **학습 운영 플랫폼**으로 발전 가능한 기반을 갖췄으며, 다음 우선순위는 **데이터 신뢰성**, **사용자 상태 영속화**, **미구현 메뉴의 제품화**입니다.
